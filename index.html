the invisible energy tax

When Sam Altman disclosed in early 2025 that the average ChatGPT query consumes 0.34 watt-hours ("about what an oven uses in a little over one second"), the internet noticed the number. What it didn't notice was the word average.

There is no routing intelligence between you and a frontier cloud model. A factual lookup and a PhD-level essay request travel the same path: from your device, through a data center, through 70-billion-parameter weights running on H100 clusters, back to you. Every query pays full freight.

The problem is not that AI is expensive. The problem is that AI does not know the difference between an easy question and a hard one.

OpenAI's own research tells the story clearly. A September 2025 study of 1.1 million ChatGPT messages found that 49% of queries are "asking", seeking information or factual advice, while only 40% are "doing" tasks requiring genuine reasoning or generation capability. Nearly half of all cloud inference spend is consumed by questions a 1-billion-parameter local model handles trivially.

classify before you compute

Academic routing systems like RouteLLM (Berkeley Sky Lab, 2024) proved that up to 85% of queries can be handled by smaller models while retaining 95% of GPT-4's answer quality. Stanford's FrugalGPT demonstrated 98% cost reduction through LLM cascading. The research was solid. Nobody had shipped it in a usable application with real energy telemetry.

SEVEN is the production deployment of that idea. Its core principle is simple: do not run inference to decide which model to run inference with. Classification happens in under 1 millisecond on the CPU, for free, before any neural network fires a single FLOP.

the routing decision costs zero energy. A keyword scan and a word-count check determine whether a query goes to a 1B local model, a real-time API, or GPT-4o in the cloud. The overhead is imperceptible. The savings compound every single query.

three tiers, one decision

Every prompt passes through a single route_prompt() call. The heuristic classifier assigns it to one of three execution tiers before any model is touched:

local - 60% of queries

0.02 Wh

Llama-3.2-1B or Phi-3.5-Mini on the AMD Ryzen AI NPU at 35 W. Average latency 412 ms. Handles factual lookups, translations, definitions.

94% energy reduction vs cloud baseline.

api / tool - 10% of queries

0.001 Wh

Direct REST calls for weather, crypto prices, breaking news. The local model synthesises the API response. No vector database. No RAG pipeline.

99.7% energy reduction vs cloud baseline.

cloud - 30% of queries

0.34 Wh

OpenAI GPT-4o, reserved for complex creative work, long-form content, and current events requiring frontier capability. Auto-escalation also lands here when the local model admits uncertainty.

the zero-cost classifier

The classifier in heuristics.py makes its decision through four ordered checks, each cheaper than inference by several orders of magnitude:

def classify_query_type(prompt: str) -> dict:
    lowered = prompt.lower()
    word_count = len(prompt.split())

    # 1. real-time keywords - always needs live data
    if any(kw in lowered for kw in REALTIME_KEYWORDS):
        return {"route": "API_CHECK", "reason": "needs_realtime_data"}

    # 2. specialised domain - beyond 1B model capability
    if any(d in lowered for d in SPECIALIZED_DOMAINS):
        return {"route": "CLOUD", "reason": "specialized_domain"}

    # 3. complexity markers - "write a detailed", "step by step"
    if any(m in lowered for m in COMPLEX_MARKERS):
        return {"route": "CLOUD", "reason": "too_complex_for_small_model"}

    # 4. length heuristic - over 150 words implies broad context
    if word_count > 150:
        return {"route": "CLOUD", "reason": "prompt_too_long"}

    # default: energy-saving local path
    return {"route": "LOCAL", "reason": "default_energy_saving"}

A post-routing validator catches failures the pre-router cannot predict. If the local model's response contains "i don't know", "i'm not sure", or "i cannot answer", SEVEN automatically retries with GPT-4o. The 3% false-negative rate costs nothing when it fires correctly, and costs one extra cloud call when it fires incorrectly, which is still cheaper than routing everything to cloud from the start.

why not just compress the model?

what this looks like at scale

ChatGPT serves approximately 2.5 billion queries per day at an average of 0.34 Wh each, roughly 850 MWh consumed daily, or 310 GWh per year for inference alone. If SEVEN-style routing were applied across 100 million users, routing 60% locally and 10% via API, the avoided inference load reaches 247.5 GWh per year, equivalent to shutting down a 28 MW power plant permanently.

Your laptop has had an NPU since 2023. Until SEVEN, nothing used it for AI inference.

SEVEN's local tier targets the AMD Ryzen AI XDNA 2 NPU, the same silicon in mid-range 2024+ laptops, drawing 35 W versus 75 W for a discrete GPU at equivalent inference throughput. It also runs on Apple Silicon via the Neural Engine and Intel Core Ultra NPU, degrading gracefully to GPU or CPU when no NPU is available.

efficiency does not have to wait for better hardware

The narrative around AI and energy tends toward inevitability: more users, more compute, more carbon, waiting for renewables to catch up. SEVEN argues for a different path. The hardware to run a 1-billion-parameter model is already in millions of pockets and backpacks. The research proving that most queries do not need frontier models is published and reproducible. The only missing piece was a system that connects these two facts at the moment a user presses Enter.

This demo saves milliwatts. At scale, it shuts down power plants.
